# Task 1.2 Bankrun Gatekeeper Tests Design

## Goal

Build a bankrun integration test suite that exercises all 10 `cloak-gatekeeper` instructions with production semantics, including successful Squads-gated paths, account realloc behavior, license close behavior, and the main negative cases for gatekeeping and payload validation.

## Context

Task 1.1 added the full instruction surface to `programs/cloak-gatekeeper/src/`:

- `init_cofre`
- `issue_license`
- `execute_with_license`
- `init_view_distribution`
- `add_signer_view`
- `remove_signer_view`
- `close_expired_license`
- `emergency_close_license`
- `revoke_audit`
- `set_operator`

The existing `tests/integration/spike-cpi.test.ts` manually encodes account data and instructions, seeds the bankrun ledger with fixture accounts, then proves `execute_with_license` can CPI into `cloak-mock`. Task 1.2 should preserve that regression and add complete instruction coverage.

## Key Technical Constraint

Most gatekeeper administration instructions require:

```rust
pub squads_vault: Signer<'info>
```

and validate that the signer is the Squads v4 vault PDA:

```rust
Pubkey::find_program_address(
    &[b"multisig", multisig.as_ref(), b"vault", &[0]],
    &SQUADS_V4_PROGRAM_ID,
)
```

A TypeScript test client cannot sign as this PDA because PDAs have no private key. The Solana runtime marks a PDA as signer only during CPI when the program that owns the PDA calls `invoke_signed` with the correct seeds. Therefore, direct TypeScript transactions can cover failure cases for missing or wrong signers, but they cannot cover successful Squads-gated instructions.

## Decision

Task 1.2 will add a local test harness Anchor program whose declared program id is the real Squads v4 id:

```text
SQDS4ep65T869zMMBKyuUq6aD6EgTu8psMjkvj52pCf
```

The harness is test-only. Its job is to expose wrapper instructions that invoke `cloak-gatekeeper` using `invoke_signed` for the Squads vault PDA. This lets bankrun execute the same signer semantics that production relies on without weakening `cloak-gatekeeper` or adding test-only bypasses.

## Architecture

### Test Harness Program

Add a small Anchor program under `programs/cloak-squads-test-harness/` with:

- `declare_id!("SQDS4ep65T869zMMBKyuUq6aD6EgTu8psMjkvj52pCf")`
- wrapper instructions for each Squads-gated gatekeeper instruction:
  - `invoke_init_cofre`
  - `invoke_issue_license`
  - `invoke_init_view_distribution`
  - `invoke_add_signer_view`
  - `invoke_remove_signer_view`
  - `invoke_emergency_close_license`
  - `invoke_revoke_audit`
  - `invoke_set_operator`
- a shared helper that builds the Squads vault signer seeds:
  - `b"multisig"`
  - `multisig.as_ref()`
  - `b"vault"`
  - `&[0]`
  - `&[vault_bump]`

The harness should not store state. It should only relay CPI calls into `cloak-gatekeeper`.

### Test Helpers

Add TypeScript helpers under `tests/integration/helpers/` for:

- PDA derivation:
  - cofre PDA
  - license PDA
  - view distribution PDA
  - Squads vault PDA
  - cloak-mock pool PDA
  - cloak-mock nullifier PDA
- Anchor discriminator calculation:
  - accounts
  - instructions
- binary encoders and decoders:
  - `Cofre`
  - `License`
  - `ViewKeyDistribution`
  - `EncryptedViewKey`
  - `StubPool`
  - `NullifierRecord`
- transaction helpers:
  - get latest blockhash
  - sign and process transactions
  - try-process expected failures and assert logs/error text
  - fund generated keypairs
  - warp/set clock for expiry tests

Helpers must validate account owner, discriminator, and expected data length before decoding.

### Test Suite

Create `tests/integration/gatekeeper-instructions.test.ts`. The suite should use `node --experimental-strip-types` like the current spike test. Task 1.2 should not introduce Vitest or Jest.

The suite should start bankrun with all Anchor workspace programs loaded:

```ts
const context = await startAnchor(ROOT, [], []);
```

The workspace must include `cloak_gatekeeper`, `cloak_mock`, and the new `cloak_squads_test_harness` program so the harness is deployed at the Squads v4 id.

## Coverage Requirements

### Initialization and License Issuance

Test successful `init_cofre` through the harness:

- cofre PDA exists
- `multisig`, `operator`, `view_key_public`, `version`, empty `revoked_audit`, and `bump` decode correctly

Test failed `init_cofre` with a wrong vault signer:

- transaction fails
- logs or error text include `InvalidSquadsSigner`

Test successful `issue_license` through the harness:

- license PDA exists
- `cofre`, `payload_hash`, `nonce`, `status = Active`, `close_authority = operator`, and `bump` decode correctly
- `expires_at > issued_at`

Test failed `issue_license` with `ttl_secs <= 0`:

- transaction fails
- logs or error text include `InvalidTtl`

### Execute With License

Keep the existing successful CPI regression:

- `execute_with_license` consumes an active license
- `License.status` changes to `Consumed`
- `cloak-mock` increments `StubPool.tx_count`
- `cloak-mock` creates the expected `NullifierRecord`

Add negative cases:

- wrong operator signer returns `NotOperator`
- expired license returns `LicenseExpired`
- consumed license returns `LicenseConsumed`
- mutated payload invariant returns `LicensePayloadMismatch`
- wrong CPI target returns `InvalidCpiTarget`

### View-Key Distribution

Test successful `init_view_distribution` through the harness:

- view distribution PDA exists
- `cofre` field matches
- entries vector is empty
- bump decodes correctly

Test successful `add_signer_view` through the harness:

- account realloc grows the data length
- one `EncryptedViewKey` entry decodes with exact signer, ephemeral key, nonce, ciphertext, and positive `added_at`

Test duplicate `add_signer_view`:

- transaction fails
- logs or error text include `SignerAlreadyExists`

Test successful `remove_signer_view` through the harness:

- target signer entry is removed
- account data length shrinks to `ViewKeyDistribution::space(remaining_entries)`

Test missing `remove_signer_view`:

- transaction fails
- logs or error text include `SignerNotFound`

### License Closing

Test successful `close_expired_license` directly from TypeScript:

- set or warp clock so `now > expires_at`
- anyone can submit the transaction
- license account no longer exists after close
- rent destination is the configured operator account

Test failed `close_expired_license` before expiry:

- transaction fails
- logs or error text include `LicenseNotExpired`
- license account remains

Test successful `emergency_close_license` through the harness:

- license account closes regardless of expiry
- rent destination is the configured operator account

### Audit Revocation and Operator Rotation

Test successful `revoke_audit` through the harness:

- cofre account realloc grows by 16 bytes plus vector length accounting
- new diversifier appears in `revoked_audit`

Test duplicate `revoke_audit`:

- transaction fails
- logs or error text include `RevocationCollision`

Test capacity failure:

- seed a cofre account with `Cofre::MAX_REVOKED` entries using `context.setAccount`
- transaction fails
- logs or error text include `RevocationCapacity`

Test successful `set_operator` through the harness:

- cofre `operator` changes to `new_operator`
- old operator can no longer execute a license
- new operator can execute a fresh license

## Error Handling and Assertions

Expected-failure tests should use `tryProcessTransaction` or a helper that catches `processTransaction` errors and checks:

- custom error name when present in logs
- custom error code text when the name is not available
- account state remains unchanged after failed transactions

The test suite should avoid brittle assertions on full log output. It should assert targeted substrings and decoded account state.

## Out of Scope

- Do not change `programs/cloak-gatekeeper/src/` behavior.
- Do not add test-only feature flags or bypasses to `cloak-gatekeeper`.
- Do not deploy to devnet.
- Do not implement `@cloak-squads/core`.
- Do not introduce Vitest or Jest in Task 1.2.
